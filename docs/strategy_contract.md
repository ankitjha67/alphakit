# Strategy contract

> This is the **non-negotiable** spec that every AlphaKit strategy
> ships with. If a PR does not meet this contract, maintainers will
> close it with a request to retrofit. No exceptions — not even for
> "experimental" strategies.

## Rationale

The value of AlphaKit is not that it has N strategies — it is that
every strategy is **audit-worthy**. A researcher should be able to
look at any strategy in the repository and, in under five minutes,
answer:

1. What paper does this come from?
2. What are the default parameters and why?
3. How does it perform on a known out-of-sample window?
4. When is it known to blow up?
5. How do I run it?

The contract below is engineered to make those five questions
answerable from the strategy's directory alone.

## Directory layout

```
packages/alphakit-strategies-<family>/alphakit/strategies/<family>/<slug>/
├── __init__.py
├── strategy.py
├── config.yaml
├── paper.md
├── benchmark_results.json
├── known_failures.md
├── README.md
└── tests/
    ├── __init__.py
    ├── test_unit.py
    └── test_integration.py
```

Every file is required. Missing any of them will fail CI via the
[PR template checklist](https://github.com/ankitjha67/alphakit/blob/main/.github/PULL_REQUEST_TEMPLATE.md).

## File-by-file requirements

### `strategy.py`

The implementation. Must:

- Define a class whose instances satisfy
  [`StrategyProtocol`](architecture.md#strategyprotocol).
- Declare class-level metadata:

  ```python
  name: str                   # unique slug matching the directory name
  family: str                 # matches the family package directory
  asset_classes: list[str]    # any subset of AssetClass values
  paper_doi: str              # DOI, arXiv id, or book ISBN — never blank
  rebalance_frequency: str    # "daily" | "weekly" | "monthly" | ...
  ```

- Implement `generate_signals(prices: pd.DataFrame) -> pd.DataFrame`.
- Be **pure**: same input → same output. No hidden RNG, no file I/O,
  no network. If the strategy genuinely needs randomness (e.g. an ML
  sampler), it must accept a `seed` argument and default it to `0`.
- Document every deviation from the published paper at the top of
  the module docstring. Rescaled volatility targets, alternative
  estimators, edge-case handling — spell it out.
- Pass `mypy --strict` with zero errors.

### `config.yaml`

A single YAML document describing the default configuration:

```yaml
strategy: <slug>
family: <family>
paper_doi: "<DOI>"

universe:
  - SYMBOL_1
  - SYMBOL_2

# Strategy parameters (named exactly as __init__ expects)
lookback_months: 12
vol_target_annual: 0.10
# ...

# Portfolio mechanics
rebalance: monthly
long_only: false
```

The `universe` key is a list of tickers that make a sensible default
demo — it is **not** a hard constraint. Users can pass any universe
they like at backtest time.

### `paper.md`

A markdown document with:

1. **Citation block** — title, authors, venue, year, DOI.
2. **BibTeX block** — copy-pasteable.
3. **Abstract verbatim** from the source.
4. **Published parameters table** showing paper defaults vs. AlphaKit
   defaults, with notes on any rescaling.
5. **In-sample / out-of-sample windows** used by the original study.
6. **At least one known replication** (book, follow-up paper, or
   practitioner blog post).
7. **Implementation deviations** list.

See the reference
[`tsmom_12_1/paper.md`](https://github.com/ankitjha67/alphakit/blob/main/packages/alphakit-strategies-trend/alphakit/strategies/trend/tsmom_12_1/paper.md)
for a template.

### `benchmark_results.json`

A JSON document that conforms to the Appendix C schema (reproduced
below). Generated by `scripts/benchmark_strategy.py` during the
nightly benchmark run; committed to the repository so that
benchmark regressions can be detected via diff.

```json
{
  "slug": "tsmom_12_1",
  "benchmark_date": "2026-04-15",
  "data_start": "2000-01-01",
  "in_sample_end": "2015-12-31",
  "out_of_sample_end": "2025-12-31",
  "universe": ["SPY", "EFA", "EEM", "AGG", "GLD", "DBC"],
  "metrics": {
    "sharpe": 0.78,
    "sortino": 1.12,
    "calmar": 0.45,
    "max_drawdown": -0.18,
    "annualized_return": 0.082,
    "annualized_vol": 0.105,
    "turnover_annual": 3.2,
    "capacity_usd_bn": 15.0
  },
  "regime_performance": {
    "bull_market_sharpe": 0.95,
    "bear_market_sharpe": 0.42,
    "sideways_sharpe": -0.15
  },
  "transaction_costs_assumed_bps": 5,
  "commit_sha": "abc123...",
  "engine": "vectorbt"
}
```

Every field is required. Unknown values must be `null`, never
omitted, and a strategy with any `null` metric field is flagged
`status: "placeholder"` in CI.

### `known_failures.md`

A prose document that **honestly** enumerates the regimes where the
strategy underperforms. Every bullet must name at least one
historical window and a qualitative Sharpe / DD expectation. Vague
hand-waving like "doesn't work in volatile markets" is not
acceptable — be specific.

### `README.md`

A one-page user guide that opens with a 10-line quickstart and then
links to `paper.md`, `config.yaml`, `known_failures.md` and
`benchmark_results.json`.

### `tests/`

At minimum:

- `test_unit.py` — exercises `generate_signals` on synthetic data.
  Must cover:
  - protocol conformance (`isinstance(strategy, StrategyProtocol)`)
  - metadata integrity (name, family, paper_doi, rebalance_frequency)
  - constructor argument validation
  - shape and dtype of the output
  - economic behaviour on hand-crafted trending / flat / reversing
    panels
  - determinism

- `test_integration.py` — runs a full backtest through at least one
  bridge (`vectorbt_bridge` is the Phase 0 default) on a synthetic
  multi-asset panel and asserts that the returned
  [`BacktestResult`](architecture.md#backtestresult) is internally
  consistent: equity curve aligns to prices, metrics are finite, max
  drawdown is non-positive.

Coverage gate: overall project coverage must stay ≥ 85%.

## Acceptance criteria for a new strategy PR

A PR is only ready to merge when **every** item is true:

- [ ] Every file in the directory layout is present.
- [ ] `strategy.py` satisfies `StrategyProtocol` (verified by
  `isinstance` in `test_unit.py`).
- [ ] `paper.md` has a non-empty DOI / arXiv link.
- [ ] `benchmark_results.json` has non-null metric values **or** is
  explicitly marked `status: "placeholder"` for Phase 0 bootstrap
  strategies.
- [ ] `known_failures.md` lists at least one honest regime.
- [ ] `ruff check` passes.
- [ ] `mypy --strict packages/` passes.
- [ ] `pytest` passes with coverage delta ≥ 85%.
- [ ] `CHANGELOG.md` updated under `[Unreleased]`.
- [ ] PR description includes the strategy summary from the PR
  template.
